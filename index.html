一些css的小技巧

content-box和border-box(IE开发的)

width = 100%时，指该元素相对于父容器的content区宽度相等。
border-box的width = content + padding + border。IE默认是border-boxing。border-box与width:100%配合。

margin合并

outline与border的区别

总结：这两者的区别有:
1.outline是不占空间的，既不会增加额外的width或者height（这样不会导致浏览器渲染时出现reflow或是repaint）
2.outline有可能是非矩形的（火狐浏览器)下。

margin合并问题

垂直 margin 默认会合并(父子之间的 margin 合并、兄弟之间的margin合并)，左右两边不合并。

![此时兄弟元素div1、div2的margin-top、margin-bottom还有child的margin-top都合并了，取最大值]

避免margin合并 1.父元素加padding-top/bottom 2.父元素加border 3.float
或者div之间有东西 也不会出现合并现象

可查文档

负的margin

在文档流中，元素的最终边界是由margin决定的，margin为负的时候就相当于元素的边界向里收，文档流认的只是这个边界，不会管你实际的尺寸是多少。

margin-left为负时，使元素向左移动。当右侧无元素时，margin-right为负无作用。有的话，则会使后面的元素向它靠拢。

当一子元素没设宽度或宽度为auto时，设置它的margin-left/right为负，会使它向左右方向扩张。

padding

padding不能为负。其它与margin属性类似，百分比值的padding属性(padding-top/buttom)参照其生成盒(父元素)的包含块的width(内容区)。

padding-top的百分比应用

不设子元素宽高，使padding-top=100%，会使该元素的高度被撑开，子元素的宽和父元素的content宽度相同，长 = padding-top = 父元素的content，所以它是一个正方形。改变百分比，可以形成不同的形状，且形状会随父元素变化。由于父元素随子元素填充的高度决定，所以二者一起变化。因为内边距被填充，所以子元素不能有内容。

span

span之间加&nbsp可以使它有高度，行盒高度由line-height决定，内敛元素永远处于行盒(不可见)的中心位置中。span的margin-top、bottom没有反应,padding-top/bottom可以使span的边界发生变化，但是不能撑开它的行高，父元素div依旧就由原来span元素的line-height决定，font-size也不影响。

将span设成inline-block，padding就会影响上span的高度，改变行高。

float

内敛元素float后，默认display：block，会脱离文档流。文档流中的块级元素会忽略浮动元素，但内敛元素会环绕它，float后不代表其position会变化。

clear：left/right/both，使该元素的左右两边不能有浮动元素，该元素会向下平移一直身边没有浮动元素。一般用于浮动元素突破边际，超出父元素，在底部设置div清除浮动，可以使父元素重新覆盖该浮动元素。可用
：after{content:’’; ##display:block##; clear:both;}来实现。内敛元素不能用clear属性。

三栏分布：先设两个左右浮动的元素，再设中间的元素。

几种位置

static:写和没写一样，与文档流保持一致，仍在文档流中。

relative:仍在文档流中，不影响其他文档流元素中的位置，距离自己位置做偏移，但依然占据原来文档流的位置。当偏移值取百分比时，是按父元素的宽高，有可能覆盖其他元素。

fixed（注意是fixed不是fix）：相对于视口(浏览器)定位,不在文档流中，其他元素会填充它的位置。偏移量写百分比时，取值是父元素的宽高。fixed的margin设为auto，可以在屏幕居中。

absolute 相对于祖先中最近的已定位的元素(已定位元素 == position不为static)定位，如果找不到，则以最外面的包含块(html)定位，定位时紧贴border，忽略padding，absolute同样脱离文档流。

FLEX

FLEX的概念

弹性盒子FLEX，是为了解决之前用传统CSS方法float，postion完成常见的垂直居中，排版布局带来的不便和晦涩难懂。新的工具flex具有快捷方便，语法易懂的特点。目前也得到广大主流浏览器的青睐。

设为Flex布局以后，子元素的float、clear和vertical-align属性将失效。

以下是弹性盒子的主要构成：

FLEX容器的属性

FLEX容器的父元素的属性

-flex-direction:column / row / column-reverse / row-reverse. (主轴方向)
-flex-wrap:nowrap / wrap / wrap. (如何换行)
-flex-flow:前两者的简写。
-justify-content:flex-start / flex-end / center / space-between / space-around.

-align-items:flex-start / flex-end / center / baseline / stretch(默认).

-align-content:flex-start / flex-end / center / space-between / space-around / stretch.

FLEX容器的子元素的属性

-order: 按顺序排列，每个子元素都必须编号。
-flex-grow: 该子元素吞并剩余空间的比例。
-flex-shrink: 按比例缩小，若都为1，则全都缩小一样的比例。
-flex-basis：该子元素占据主轴方向的多少。auto为原本大小，设为与长或宽一样时，占据整个空间。
-flex:前四种属性的简写。
-align-self:该子元素有与其他不一样的对齐方法，覆盖align-items。
